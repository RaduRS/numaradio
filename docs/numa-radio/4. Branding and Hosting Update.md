The project will be called **Numa Radio**.

The main public domain will be **numaradio.com**.

The public website/frontend will be hosted on **Vercel**.

The **NanoClaw** backend runs **locally on a dedicated always-on mini-server** at home (used purely for the website — not the user's day-to-day laptop).

Because a live public website cannot directly access a local network service, the local NanoClaw service is exposed through **Cloudflare Tunnel**. Cloudflare Tunnel was chosen over alternatives (ngrok, localtunnel, etc.) for being the most reliable option with a usable free tier.

The planned structure is:

- `numaradio.com` → public website/frontend on Vercel
    
- `api.numaradio.com` → Cloudflare Tunnel → local NanoClaw service running on the dedicated mini-server
    

This setup is suitable for MVP. The mini-server is intended to remain powered on and running NanoClaw continuously. Migration to managed cloud hosting (e.g. Fly, Railway, Hetzner, AWS) is a planned later step once the product is stable.
